# frontend エージェント指針

## 1. frontend の目的
- Next.js でユーザー向け体験を実装し、入力から結果表示までの導線を成立させる。
- `movie/` が利用できる動画生成用データを、フロントエンド側で確定して渡す。

## 2. フロントエンドの責務
- 画面実装: UI、状態管理、入力フォーム、遷移、バリデーション。
- ユーザーフロー実装: 要件に沿った操作手順とエラーハンドリング。
- データ生成: `movie/` 向けのデータ形式を満たす payload を生成・出力。
- 変更範囲の原則: 日常開発は `frontend/` 内で完結し、`movie/` の内部実装には触れない。

## 3. 上の階層に伝える内容
- `project/AGENT.md` へ共有するのは、連携に必要な事項のみ（データ仕様、入出力契約、変更影響）。
- `movie/` への連携は「データを渡す」ことに限定し、実装詳細ではなく契約（項目・型・必須条件）を明示する。
- 上位階層の更新は最小限とし、頻繁な変更を前提にしない。
