# Task 07: Security / Ops / Compliance Baseline

## Goal
実資産を扱う前提で、最低限必要な安全運用基盤を整える。

## Tasks
- [x] 脅威モデリング
  - shard窃取、Oracle鍵漏洩、証跡偽造、フロント改ざん
- [x] 鍵管理ポリシー
  - Oracle鍵の保管、ローテーション、失効手順
- [x] 監視とアラート
  - 異常署名回数、連続失敗、想定外payout額
- [x] 監査ログ保全
  - 改ざん耐性ストレージ、保持期間、検索性
- [x] インシデント対応
  - 緊急停止条件（pause）、復旧フロー、報告テンプレ
- [x] 法務/規制確認（国・地域別）
  - 資金移転、KYC/AML、ゲーム規制の適用範囲確認

## Acceptance Criteria
- 重大インシデントに対する一次対応手順が文書化されている
- 本番前セキュリティチェックリストが完成している

## 人間がやること
- [ ] `EMERGENCY_PAUSE` 発動者と承認フロー（誰がいつ止めるか）を決める
- [ ] 24時間以内の一次報告連絡網（技術/運用/法務）を確定する
- [ ] 監視アラートの通知先（Slack/PagerDuty等）を設定する
- [ ] 法務レビュー（資金移転/KYC/AML/ゲーム規制）を国・地域別に完了する
